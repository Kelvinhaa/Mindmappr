# MindMappr

An AI-powered study assistant that generates personalized study recommendations based on your subject, available time, and expertise level. Built with FastAPI and vanilla JavaScript, powered by Anthropic's Claude API.

## Deployment

- Vercel: mindmappr-omega.vercel.app
- AWS Elastic Beanstalk (Docker platform): MindMappr-env.eba-gkfhscmg.ap-southeast-2.elasticbeanstalk.com 

## Usage

1. Enter the subject you're studying
2. Set your available study duration (in minutes)
3. Select your expertise level (Beginner / Intermediate / Advanced)
4. Click Get Recommendations and receive a personalized study plan

## Introduction

MindMappr helps students and learners optimize their study sessions by providing tailored techniques and strategies. Simply enter what you're studying, how long you have, and your current level — the app returns actionable study recommendations generated by Claude AI.

## Tech Stack

Backend
- Python 3
- FastAPI
- Anthropic SDK (Claude API)
- Pydantic (data validation)
- Uvicorn (ASGI server)

Frontend
- HTML5 / CSS3 / JavaScript (vanilla, no frameworks)
- Responsive dark theme with CSS custom properties

## Installation

### Prerequisites

- Python 3.10+
- An Anthropic API key from https://console.anthropic.com/

### Backend

```bash
cd backend

# Create and activate virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env and add your Anthropic API key:
# ANTHROPIC_API_KEY=your_key_here

# Start the server
python -m backends.main
```

The API will be available at http://localhost:8000.

### Frontend

Serve the frontend/ directory with any static file server:

```bash
cd frontend

# Option 1: Python
python -m http.server 5500

# Option 2: VS Code Live Server extension

# Option 3: Node
npx http-server -p 5500
```

Then open http://localhost:5500 in your browser.

## AWS Backend Deployment (Elastic Beanstalk)

The backend is deployed to AWS Elastic Beanstalk using a Docker-based runtime.

1. Create an Elastic Beanstalk application and environment.
2. Use Docker platform for the environment.
3. Confirm environment health is Green before first CI/CD deploy.

## CI/CD with GitHub Actions

Pipeline file: .github/workflows/aws-ci-cd.yml

Behavior:
- Pull requests and backend changes run CI checks.
- Pushes to main run CI and then deploy to Elastic Beanstalk.
